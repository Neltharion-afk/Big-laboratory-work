# Матрица доступа (ACL)

## Правила разграничения доступа

| Откуда ↓ / Куда → | Mgmt (10) | Sales (20) | IT (30) | Servers (40) | DMZ (50) | Guest (60) | Интернет | Филиал |
|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| **Mgmt (10)** | + | + | + | + | + | + | + | + |
| **Sales (20)** | - | + | - | + | + | - | + | + |
| **IT (30)** | + | - | + | + | + | - | + | + |
| **Guest (60)** | - | - | - | - | - | + | + | - |

## Реализация

ACL применены на SVI ядер (CORE-1, CORE-2):

**ACL_SALES (VLAN 20):**
- Запрет к Management (VLAN 10)
- Запрет к IT (VLAN 30)
- Запрет к Guest (VLAN 60)
- Разрешено всё остальное

**ACL_IT (VLAN 30):**
- Запрет к Sales (VLAN 20)
- Запрет к Guest (VLAN 60)
- Разрешено всё остальное

**ACL_GUEST (VLAN 60):**
- Разрешён DNS к серверу
- Запрет ко всем внутренним сетям
- Разрешён Интернет

## Примечание

В Cisco Packet Tracer ACL на SVI требуют точной привязки командой `ip access-group`. 
На реальном оборудовании работают стабильно.
